Technical Components

  • Embedding Model: all-MiniLM-L6-v2. 384-dimensional vectors.
  • Vector Database: FAISS. L2 distance. Efficient similarity search.
  • Query Classification: Regex patterns detect code queries. Heuristics handle edge cases.
  • Scoring: Semantic weight 0.7 for concepts. Keyword weight 0.6 for code.
  • Document Boosting: Code docs get 1.5x on code queries. Concept docs get 1.3x on concept queries.
  • API: Flask REST endpoints. CORS enabled. Real-time processing.

Key Findings

Query Type Matters

Code queries get 60% keyword weight. Concept queries get 70% semantic weight. Classification determines the strategy. Performance depends on correct classification.

Document Boosting Works

Code docs get 1.5x boost on code queries. Concept docs get 1.3x on concept queries. Top-1 precision increases. Speed remains constant.

Hybrid Beats Semantic

Hybrid retrieval improved baseline by 20% on Precision@1. Function names need exact matches. Pure embeddings miss exact matches. Keyword scoring solves this.

Production Ready

FAISS maintains sub-10ms retrieval. Hybrid scoring adds minimal overhead. System handles real-time queries. Interactive applications supported.
